
Iterative Prompt Mutation for Multi-Step AI Agent Red Teaming
Author: Sahaj Gupta
GitHub Repository: https://github.com/PhenomSG/ai-red-teaming

Abstract
An investigation of iterative prompt mutation as a systematic technique to discover reproducible multi-step failures in tool-using AI agents. The work emphasizes reproducibility, deterministic replay, and strategies that influence planning, tool invocation, and execution flow over long-horizon interactions.

Keywords: LLM agents, prompt mutation, red teaming, tool use, deterministic replay

1. Introduction
Large Language Model (LLM) agents increasingly plan across multiple steps, call external tools, and autonomously execute tasks. While these capabilities improve utility, they also expand the attack surface: adversarial instructions can manipulate reasoning, tool selection, and execution order. Existing jailbreak work often treats prompts in isolation and overlooks long-horizon, multi-step interactions. This motivates a systematic evaluation of autonomous agents under realistic adversarial settings.

2. Methodology
- Iterative prompt mutation: prompts are mutated based on prior agent behavior rather than handcrafted jailbreaks.
- Replayability: interactions are designed to be deterministic so successful attack trajectories are reproducible.
- Mutation targets: reasoning chains, tool selection heuristics, invocation order, and intermediate conversational state.
- Search strategy: balance exploration (discover new adversarial trajectories) with exploitation (refine promising prompt patterns).

3. Current Observations
- Agent behavior depends on more than prompt text: tool ordering, intermediate reasoning state, and accumulated context all matter.
- Some adversarial trajectories are reproducible across repeated runs, indicating vulnerabilities that survive stochasticity.
- Evaluating safety using isolated prompts is insufficient for tool-using, long-horizon agents.

4. Future Work
- Develop richer behavioral metrics to measure robustness across multi-step interactions.
- Automate evaluation pipelines for large-scale, repeatable red-teaming experiments.
- Build benchmarks that capture long-horizon reasoning, tool use, and stateful interactions.
- Study mitigation strategies that make agents resistant to reproducible multi-step attacks.

5. Conclusion
Iterative prompt mutation offers a principled way to explore the adversarial space for multi-step AI agents. By emphasizing reproducibility and targeted mutation strategies, this approach helps surface systematic failure modes that isolated prompt tests miss. The next steps are metricization, automation, and community benchmarks to better assess and improve agent robustness.
